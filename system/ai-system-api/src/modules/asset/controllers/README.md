## 3. 资产实例绑定方

- **操作方**：出租方（商家 / 个人）
- **操作前置条件**：
  - 订单状态 = `待发货`
  - 资产模板归属当前出租方
  - 资产实例状态 = `可用`

## 5. 绑定规则

### 5.1 绑定约束

- 一个订单 **只能绑定一个实例**
- 一个资产实例 **同一时间只能绑定一个订单**
- 资产实例必须：
  - 归属于订单资产模板
  - 状态为 `可用`
- 绑定要有记录 
  `asset-inventory-rental-record.entity.ts` 记录表

### 5.2 状态变更

绑定成功后：

| 对象 | 状态变化 |
|----|----|
| 资产实例 | `可用` → `租赁中` |
| 资产模板 | 不变 |

## 6. 业务流程说明

### 6.1 出租方操作流程

1. 出租方进入订单详情页
2. 系统提示「请绑定资产实例后提醒用户收货」
3. 出租方从可用实例列表中选择实例
4. 确认实例信息（SN / 编号）
5. 提交绑定请求
6. 系统校验并完成绑定
7. 允许填写物流信息并发货

---

### 6.2 系统校验逻辑

- 校验订单状态是否合法
- 校验实例归属关系
- 校验实例当前状态
- 校验实例未被其他订单占用

---

## 7. 异常与边界场景

### 7.1 绑定后取消订单

- 出租方取消订单或者同意用户申请的取消订单：
  - 解绑资产实例
  - 实例状态回退为 `可用`
---

### 7.2 绑定错误

- 未发货前允许重新绑定
- 重新绑定需先解绑原实例
- 所有操作记录审计日志

---

### 7.3 多实例资产

- 同一资产模板可存在多个实例
- 实例之间互不影响
- 系统仅在绑定时锁定实例

## 9. 风控与扩展（预留）

### 9.1 风控能力

- 实例绑定后不可随意更换
- 异常频繁更换实例的出租方触发风控
- 高价值资产强制绑定实例

---

### 9.2 扩展能力

- 支持二维码 / NFC / RFID 绑定
- 支持链上实例哈希存证
- 支持 IoT 状态上报
